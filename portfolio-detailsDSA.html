Project information

  • Category: Game Development - Internship
  • Project date: 2025
  • Project Engine: Unity
  • Project Type: Unity Information App

Defense Simcenland App Page

I developed a Unity-based app designed to inform new interns about the location and details of the internship, including the rules, regulations, and workflow of the branch they would be working in.


Checklist Selection Page

The Checklist selection page was created to guide new interns through the process of submitting monthly salary-related documents and applying for transportation reimbursement..


Type of Checlist

This page was designed to allow the user to select the type of checklist they wanted to create.



Checklist Page

This page appeared after the user selected one of two options and was designed to guide new interns through the process of completing a reimbursement form for transportation to the internship location.


Quiz Selection Page

The quiz selection page in the app allowed users to choose which quiz to take and view their highest score up to that point, helping them identify areas where they needed the most improvement.

Quiz

The user would then select answers based on predetermined questions. It was a standard quiz format, with the possibility of having two to four answer choices and accompanying images.



Quiz System

The system was designed to be dynamic, allowing the company for which the app was developed to reuse it without needing any modifications. Both the answer choices and quiz options could be added dynamically through the Unity Inspector.


Results System

This system integrates with the quiz by checking answers without saving them, then deletes temporary data and only records the total correct out of all questions.

Results

The results page was designed to be streamlined, focusing on displaying the proper information, such as the Question ID, result, and review buttons.



Review Question Page

The review system was designed simply to return to the quiz and reload a question’s data, highlighting both the player’s answer and the correct answer. This allows the user to see what they got wrong as well as understand why.
